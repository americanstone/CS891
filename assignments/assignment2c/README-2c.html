Image Crawler Parallel Stream App (Assignment 2c)

The purpose of this assignment is to deepen your experience using Java functional programming features and their integration with Java parallel streams. In particular, in this assignment you'll develop an app that uses a more advanced Java parallel streams implementation to obtain, transform, and store images.

The code you'll write for this assignment has a GUI and runs as an Android app. Please feel free to use this app to help debug your solution.

Assignment Structure and Functionality

The ImageCrawler app is packaged as a project using the latest version of Android Studio. This app is written in both Kotlin and Java and demonstrates many Android capabilities. For the purposes of Assignment 2c, however, you only need to be aware of the following three directories:

  • image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils, which contains the skeleton(s) you'll need to fill in, as described below.
  • app/src/test, which is a set of JUnit tests that exercise the features under test and can be used to evaluate whether you've correctly satisfied this assignment's requirements.
  • app/src/androidTest/java/assignmentTests/assignment2cTests, which is an Android Studio instrumentation test that runs your app automatically.

To compile this code you need to use the provided Android Studio project. You can run this project by clicking the green "Run 'app'" button in the Android Studio IDE, which should automatically select an Android Emulator to run, assuming you have one created. The application's minimum API is 26, so you will need to select an emulator that uses an API >= 26. If you do not already have one created you can create it by clicking on the "AVD Manager" button in the Android Studio IDE.

Program Description and "TODO" Tasks

In this part of the assignment you'll complete the implementation of an image crawler class in the ParallelStreamsCrawler2.java file in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers folder. This classes use a more advanced Java parallel stream implementation to perform an "image crawl" starting from a root Uri to obtain, transform, and store images. Depending on the parameters used to run the tests, images from HTML page reachable from the root Uri are either downloaded from a remote web server or read from the local file system and the results are stored in files.

The bulk of the code for the app is provided in the skeletons. You'll need to complete the "TODO" tasks in the ParallelStreamsCrawler2.java file, which provides a variant of the ParallelStreamsCrawler1.java file that uses parallel streams in a more sophisticated manner.

Your solution should use no loops in this assignment and instead use Java parallel stream operations.

Unit Tests

The JUnit unit tests in the image-crawler/src/test/java/edu/vanderbilt/imagecrawler folder are provided to increase our confidence that your implementation is working as expected. You'll need to learn how to use the Android Studio GUI to run the JUnit test locally on your computer, as described here. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the JUnit tests to detect problems in your code.
