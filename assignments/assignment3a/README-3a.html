Image Crawler Fork-Join Pool App (Assignment 3a)

The purpose of this assignment is to deepen your experience using Java 8 functional programming features and their integration with the Java fork-join pool. In particular, in this assignment you'll develop an app that uses the Java fork-join pool framework to obtain, store, and transform images.

Like the earlier assignments, the code you'll write here will have GUI and will run as an Android app. Please use this GUI along with the JUnit tests discussed below to help test and debug your solution.

Assignment Structure and Functionality

The ImageCrawler app is packaged as a project using the latest version of Android Studio. This app is written in both Kotlin and Java and demonstrates many Android capabilities. For the purposes of Assignment 3a, however, you only need to be aware of the following three directories:

  • image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers, which contains the skeleton(s) you'll need to fill in, as described below.
  • app/src/test, which is a set of JUnit tests that exercise the features under test and can be used to evaluate whether you've correctly satisfied this assignment's requirements.
  • app/src/androidTest/java/assignmentTests/assignment2cTests, which is an Android Studio instrumentation test that runs your app automatically.

To compile this code you need to use the provided Android Studio project. You can run this project by clicking the green "Run 'app'" button in the Android Studio IDE, which should automatically select an Android Emulator to run, assuming you have one created. The application's minimum API is 26, so you will need to select an emulator that uses an API >= 26. If you do not already have one created you can create it by clicking on the "AVD Manager" button in the Android Studio IDE.

Program Description and "TODO" Tasks

In this part of the assignment you'll complete the implementation of an image crawler class in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/ForkJoinCrawler1.java file that obtains, stores, and transforms images. This class uses Java fork-join pool framework to perform an "image crawl" starting from a root Uri. Depending on the parameters used to run the tests, images from HTML page reachable from the root Uri are either downloaded from a remote web server or read from the local file system and the results are stored in files.

The bulk of the code for the app is provided in the skeletons. In fact, a complete solution using Java 7 features provided in the app/src/main/java/edu/vanderbilt/crawlers/SequentialLoopsCrawler.java file. You will focus on converting this Java 7-based class to the ForkJoinCrawler1 class that uses Java fork-join framework. Feel free to use Java functional programming features (such as lambda expressions and method references) and sequential streams (but not parallel streams) in your ForkJoinCrawler1.java implementation.

Unit Tests

JUnit unit tests in the image-crawler/src/test/java/edu/vanderbilt/imagecrawler folder are provided to increase our confidence that your implementation is working as expected. You'll need to learn how to use the Android Studio GUI to run the JUnit test locally on your computer, as described here. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the JUnit tests to detect problems in your code.

Concluding Remarks

The fork-join pool framework in Assignment 3a is designed to give you experience programming directly to the Java fork-join pool framework APIs. Although this framework is powerful, you'll have to keep track of various details manually (in contrast, these details are handled automatically by the Java parallel streams framework). Therefore, please start the assignment soon and ask questions in class, office hours, and on the discussion forums. The app, image crawler skeletons, and JUnit tests are extensive, though you don't need to understand them all to complete your solution.
