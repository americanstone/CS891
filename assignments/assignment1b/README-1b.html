Array (Assignment 1b)

The objective of the second part of this two-part assignment is to deepen your experience using Android Studio to develop and test an implementation of an Array interface written using Java's object-oriented and functional programming features. It's important that your Array implementation work properly since variants of it are required by all subsequent assignments in this course.

The code you'll write for this assignment has a GUI and runs as an app. Please use this GUI to help debug your solution.

The following resources may be helpful in completing this assignment:

  • java.utils.Arrays documentation.
  • The Oracle Java Tutorial

Getting Started

Before you start with Assignment 1b you'll need to do the following things:

  • Make sure that you've installed Java 8. Instructions for doing this are available here.

  • Complete your implementation of assignment 1a and incorporate any necessary changes into your assignment 1b solution based on my review of your assignment 1a submission.

Assignment Structure and Functionality

The ImageCrawler app is packaged as a project using the latest version of Android Studio. This app is written in both Kotlin and Java and demonstrates many Android capabilities. For the purposes of Assignment 1b, however, you only need to be aware of the following three directories:

  • image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils, which contains the skeleton(s) you'll need to fill in, as described below.
  • app/src/test, which is a set of JUnit tests that exercise the features under test and can be used to evaluate whether you've correctly satisfied this assignment's requirements.
  • app/src/androidTest/java/assignment1bTests, which is an Android Studio instrumentation test that runs your app automatically.

To compile this code you need to use the provided Android Studio project. You can run this project by clicking the green "Run 'app'" button in the Android Studio IDE, which should automatically select an Android Emulator to run, assuming you have one created. The application's minimum API is 26, so you will need to select an emulator that uses an API >= 26. If you do not already have one created you can create it by clicking on the "AVD Manager" button in the Android Studio IDE.

Program Description and "TODO" Tasks

An Array is an abstract data type (ADT) implemented via a single contiguous buffer that provides operations like get(), set(), add(), remove(), and size(). Your task is to implement a generic Array class in Java. This class will be different from Java built-in arrays in various ways, e.g., it can be resized and used in conjunction with other Java Collection Framework classes. Your task in Assignment 1b is to implement a generic Array class in Java that includes methods (such as replaceAll() and forEach()) using Java 8 features, such as functional interfaces.

You'll need to modify several files containing the skeleton Java code by completing the "TODO - You fill in here" tasks to provide a working solution. DO NOT CHANGE THE OVERALL STRUCTURE OF THE SKELETON - just fill in the "TODO" tasks!!! In particular, you'll need to finish implementing the following "TODO" tasks for this assignment in the utils folder:

  • UnsynchronizedArray.java. You'll need to complete the "TODO" tasks in various methods to implement an Array interface above and beyond what you did for assignment 1a.
  • Assignment.java. You'll need to complete the "TODO" to set the sTypes field to either GRADUATE or UNDERGRADUATE, depending on which version of the assignment that you're implementing.

Graduate students (or students who are taking this course for graduate credit) must do the following additional work:

  • Implement the requirements of the Spliterator interface. Please see the videos here and here for more information on Java Spliterators.

You should avoid using loops in this assignment as much as possible and instead use Arrays.copyOf() and/or System.arraycopy() (this isn't possible for all parts of the code, of course).

Unit Tests

JUnit unit tests in the image-crawler/src/test/java folder are provided to increase our confidence that your implementation is working as expected. You'll need to learn how to use the Android Studio GUI to run the JUnit test locally on your computer, as described here. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the JUnit tests to detect problems in your code.
